# 序

需要注意的是ECS并非是unity的ECS，而是一种编程思想



# 架构

## Component

要理解 Anywhere Component 是一非常重要的点，它类似于 ECS中的 S,但它除了调度其下所有 Enity之外，也会负责调用其创建的子 Component，其第一个由框架创建的 RootComponent 是所有 Component的父节点，也是超级Root节点

## Controller

Controller 需要注意的是它是基于内嵌编程思想的设计，用于提供外部 Hooker 修改 Component 或者是Entity 的默认行为逻辑

## Enity

在Anywhere中所有可见实体都可以认为是一个`Entity`,一小部分不可见全局的也可以算作 Entity


## Static

如其名，代表全局的辅助类.
